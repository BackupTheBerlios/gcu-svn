GCU Bicycle Drive Train Calculator V. 1.0 beta 1

Computes transmissions and geometry for derailleur-geared bicycles.

Overview

Another gear calculator! Most of you might say. And, actually, that's exactly what it is! This one does, among other things:

  • Draw a scale graphic of the drive train
  • Compute chainline characteristics according to user-defined limits
  • Allow the user to load and save drive train profiles
  • Platform independent, offline use
  • Doesn't depend on third-party software other than the Java Runtime Environment (JRE)
  • Licensed under "GNU General Public License"

This thingy is the result of many hours of train riding during the past few cold and rainy days where I was either too lazy or to sick to go to my daily work by bike. I hope it will be useful...

Have fun!

Getting Started

This application uses the JRE 5.0 (1.5.0) or higher, available from http://java.sun.com/j2se/1.5.0/download.jsp. Chances are that it is already installed on your computer, so, before dowloading, check if you have it already.

There are no other installation activities required on your side.

Starting with Windows Explorer or other graphical file manager

When you are sure the JRE has been installed correctly, launch the file "gcu.jar" in the usual way you launch applications in your file manager.

After a few seconds, the GCU main window should show up.

Starting from a console

Change to the directory where the "gcu.jar" file is located, and exeute this command:

java -jar gcu.jar <profile-file-name>

profile-file-name is the optional path of a file that has been saved by GCU before. The file name usually ends with ".rrp"

If you omit this argument, GCU starts with a new empty profile.

Usage

Input Form

On the left in the main window are the input controls that must be filled with the drive train's data.

GCU has two different aspects of gear calculation: Parameters that affect translation, development (gear inches), and speed, and parameters that affect the graphical representation and chainline quality.

These aspects are seperated on two tabbed panes that can be filled in independently of each other.

The two tabbed panes themselves are divided into three sections: Chainwheels, Sprockets, and General Settings.

Tab "Translation"

Chainwheels

Here you enter the number of teeth for the front chainwheels (chainrings, front sprockets). Although possible otherwise, you should enter the chainring tooth counts in descending order to reflect the configuration of a real-world bicycle.

You can remove a single row by clicking on the trash can icon. If something is entered into the empty row at the bottom of the input table, a new chainring is added. All changes are reflected immediately in drawing and output table after you leave the input field.

Sprockets

This is for entering the parameters for the rear sprockets. Here goes the same as above under "Chainwheels", with the notable exception that you now should enter the tooth counts in ascending order.

As soon as you have entered enough data, all possible gear combinations are shown in the drawing and output table (read on for details on these).

General

Here you enter either the wheel diameter or the wheel circumference in the unit shown right of the input field, it doesn't matter which one you enter, the other one will be computed and shown immediately.

Additionally, you can enter a cadence (pedalling frequency), which will be used to compute the speed of the bicycle in the different gears.

Tab "Geometry"

Chainrings

Enter the chainline and the chainring spacing of your crankset here.

The chainline is the distance from the plane defined by the bicycle frame's forward and upward axes to the center of the distance between the largest and the smallest chainring. If this sounds a bit complicated, just have a look to the graphic, and hopefully things will become clearer.

The chainring pitch is the distance between the center of one chainring and the center of the next, and is used for the chainline computations and shown in the graphical representation.

Sprockets

Here, again, this is similar in spirit to the chainring settings. Unfortunately, measuring the rear chainline is not really as easy as measuring it in the front, but using an old rear axle with an orthogonally attached steel rod, you could project the frame center's position onto the rear axle and then measure the distance to the middle of the rear sprocket block. Another method to measure the rear chainline is explained by Sheldon Brown (http://www.sheldonbrown.com/gloss_ch.html#chainline)

General

Enter the Rear Center of your bike here. The Rear Center is the distance from the center of the crank axle to the center of the rear axle.

We need this value for chainline computations and for the drawing.

Under "Chain Straightness" you can enter two tolerance values (in degrees) for the maximum angle that the chain may be bent by to be considered a certain condition.

All chainlines bending the chain at an angle lower than the "optimal" limit are displayed in the drawing and output table as green lines. An angle between the "optimal" and the "acceptable" limit will show up in yellow, and everything above the "acceptable" limit will be marked in red.

Menu / Toolbar "View"

You can filter the gears shown in the drawing and the output table according to their chainline condition.

There are three options:

  • Only optimal gears
  • Acceptable and optimal gears
  • All gears

This feature should increase readability in case you are only interested in the gears you would actually use. By default, all gears are shown.

Drawing

The drawing is a top view of the drive train. The frame centerline is depicted by a dash-dotted line.

The measures shown on the drawing represent the values entered in the "Geometry" form.

You can adjust the zoom factor with the slider control. Panning by clicking and dragging the mouse on the drawing area.

If you have a wheel mouse, you can also use the mouse wheel to adjust the zoom factor.

Output Table

Here, the results of the computations are shown. The development (or gear inches) are visualized by a colored bar, the color of which represents the chainline status according to the rules explained above.

The table rows can be sorted by clicking the header cells according to three different criteria: Chainwheel/sprocket size, translation descending, and translation ascending.

The small arrow icon in the table header depicts the kind of ordering currently in effect.

By clicking on a row, you can select a gear that will be shown highlighted in the drawing. Entering new values, changing the window size, or changing the table sorting order clears the selection and shows all visible gears in the drawing again.

"Options" Menu

Metric Units / Imperial Units

GCU can compute values in two different measuring systems, namely the metric system (SI system), or the anglo-american system (imperial / customary units).

Chosing a unit system ot only affects the pure unit representation of values entered by the user or displayed by the application, it also makes a difference whether the Development in meters or the Gear Inches are shown in the output table.

Gear Inches are a common notion in countries that use inch-based measuring systems, this value is the translation factor multiplied by the rear wheel diameter in inches.

The Development in meters is displayed if the metric system is in effect. The development value represents the distance traveled with one full crank round. It is calculated as the wheel circumference in meters multiplied by the translation ratio.
